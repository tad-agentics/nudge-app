# Build Plan — Nudge

> Generated by the Tech Lead during `/setup`. Agents follow waves in order unless marked **PARALLEL**.
> Sources: `northstar-nudge.html` (v4) §3 / §7 / §12, `CHANGELOG-compiled.md`, `tech-spec.md`, `screen-specs-nudge-v1.md`, `emotional-design-system.md` (v4), `app/` + `src/make-import/`.

**Northstar v4 — plan invariants (do not regress):**

- **Freemium-first:** 5 active tasks + Do next on free; calendar scheduling, weekly review, unlimited tasks, Gmail-heavy flows → **paid** (northstar §4).
- **Reasoning:** batch **nine mental models** + validation gate + template fallback; **manual reorder / energy / Do first** → **`Follow-on`** table (not blocking waves 0–7).
- **Morning plan:** **Preview → approve → write**; **approved day locked** on calendar; **Do next** still **re-ranks** on task change; **today-urgent** inserts use **confirm** path only (northstar §3e, tech-spec TD-4).
- **Ambient execution (target):** calendar + notifications should preference **action surfaces** (mailto, https, tel); PWA Phase 1 may still use **`/app/*`** deep links where the platform requires it (northstar §3, tech-spec §16).
- **Phase 2 (out of scope here):** Apple Calendar, native push/widgets, voice — see northstar §12 feature grouping.

## Wave 3 — Core engine & Do next (`core-engine`)

### Backend

- `tasks` table RLS; active task count check (≤5 free) in Edge or RPC before insert.
- `llm-engine`: `structure`, `reason_batch`, **`weekly_insight`** (ship in **wave 6** with WeeklyReview), `decompose` (defer or stub); output schemas + validation gate (TypeScript shared if possible); log gate rejections for **prompt refinement loop** (northstar §3b).
- `behavioral_events`: insert on surfaced/completed/skipped per northstar §3d.

### Frontend

- **Make source:** `DoNextScreen.tsx`  
- **Components (from spec):** `TaskCaptureInput`, `DoNextCard` (4px top stripe if `skip_count >= 3`), `TaskRationale`, `ScheduledTimeChip`, `StartButton`, `DoneButton`, `SkipButton`, `TaskInboxList`, `DecompositionModal`, `ConnectGoogleModal`, `FreeLimitModal`, `SkipToast`, recurring + NL edit + push prompt (per screen spec deltas).  
- **Data vars:** `tasks.*`, `profile.*`, `limits.*` — hooks in `app/hooks/`.  
- **Copy slots:** `input_placeholder`, `section_do_next`, `task_done*`, `skip_toast`, `auth_prompt_*`, `decomposition_offer`, `free_limit_body`, `nav_*`, `recurring_created`, `edit_confirm_*`, `push_permission_rationale`.  
- **States:** loading (structuring inline), error, earned empty `no_tasks`, first_open placeholder.  
- **Dopamine:** **D1, D2, D4** on Done sequences (EDS §6) — `motion` + 1.5–2.5s pause.
- **v4 incomplete-list behavior:** users do not need a fully captured backlog — engine **re-ranks on every state change**; after morning approve, **calendar day stays locked** but **Do next** can jump to a newly added higher-ranked task until **today-urgent** flow confirms a schedule change (northstar §3e, tech-spec TD-4).

### Acceptance

- Capture → structure → reason → top task; Done/Skip update DB + skip stripe UI; freemium modal at 5 actives.
